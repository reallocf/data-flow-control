# One policy per line. Lines starting with '#' are ignored.
#
# DFCPolicy format:
#   SOURCES <source1, source2> [SINK <sink>] CONSTRAINT <expr> ON FAIL <REMOVE|KILL|INVALIDATE|INVALIDATE_MESSAGE|LLM> [DESCRIPTION <text>]
#
# Notes:
# - All columns in constraints MUST be qualified (e.g. transactions.revenue).
# - If you reference a source-table column, it must be inside an aggregate (e.g. max(transactions.revenue)).
#   For non-aggregation queries, the rewriter will transform aggregates to their underlying columns.
#
# REMOVE: filter out violating rows
SOURCES transactions CONSTRAINT max(transactions.amount) < 500 ON FAIL REMOVE DESCRIPTION Limit large amounts

# REMOVE: filter budget rows over 1000
SOURCES budgets CONSTRAINT max(budgets.amount) < 1000 ON FAIL REMOVE DESCRIPTION Budget under 1k

# KILL: abort query if any account has negative balance (credit card)
SOURCES accounts CONSTRAINT min(accounts.balance) >= -2000 ON FAIL KILL DESCRIPTION Block deep negative balance

# KILL: abort query if category count exceeds threshold
SOURCES categories CONSTRAINT max(categories.id) <4  ON FAIL KILL DESCRIPTION Limit category scope
